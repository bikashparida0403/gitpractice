=========================================================================
SECTION 1 — SOFTWARE TO DOWNLOAD & INSTALL FIRST
=========================================================================

The following tools must be installed before anything else.
Download links are listed beside each tool.

1. Node.js (v20 or higher)
   → https://nodejs.org/en/download
   → Choose "LTS" version (Windows Installer .msi)
   Dummy output after install:
     > node --version
     v20.18.0

2. Python (v3.11 or higher)
   → https://www.python.org/downloads/
   → Check "Add Python to PATH" during install!
   Dummy output after install:
     > python --version
     Python 3.11.9

3. pnpm (Package Manager for Node)
   → Install AFTER Node.js via terminal:
     > npm install -g pnpm@9.15.0
   Dummy output:
     > pnpm --version
     9.15.0

4. Git
   → https://git-scm.com/download/win
   Dummy output after install:
     > git --version
     git version 2.47.0.windows.1

5. Ollama (local AI model runner)
   → https://ollama.com/download
   → Choose "Windows" version
   Dummy output after install:
     > ollama --version
     ollama version 0.6.2

   After installing, pull the models (run these once):
     > ollama pull llama3.1:8b
     > ollama pull nomic-embed-text

   Dummy output:
     pulling manifest...
     pulling llama3.1:8b... ████████████ 100% 4.7 GB
     pulling nomic-embed-text... ████████████ 100% 274 MB
     success

   NOTE: llama3.1:8b is ~4.7 GB download. nomic-embed-text is ~274 MB.
   Ollama runs silently in the background on port 11434.

6. Redis (for rate limiting/caching)
   → On Windows, use WSL2 or Docker (recommended):
     Option A - Docker Desktop: https://www.docker.com/products/docker-desktop/
     Option B - Redis on WSL: Enable WSL2 from Windows features first.
   
   Quick Redis via Docker:
     > docker run -d -p 6379:6379 --name redis redis:alpine
   
   Dummy output:
     > docker ps
     CONTAINER ID   IMAGE          COMMAND                  STATUS
     a1b2c3d4e5f6   redis:alpine   "docker-entrypoint.s…"   Up 5 seconds

7. VS Code (optional but recommended)
   → https://code.visualstudio.com/download

SECTION 6 — SET UP PYTHON VIRTUAL ENVIRONMENT (for the API)
=========================================================================

Navigate to the API folder:

  > cd apps\api

Create a virtual environment:

  > python -m venv venv

  Dummy output:
    (no output, creates a "venv" folder)

Activate the virtual environment:

  On Windows PowerShell:
  > .\venv\Scripts\Activate.ps1

  On Windows CMD:
  > .\venv\Scripts\activate.bat

  Dummy output after activation:
    (venv) PS C:\Projects\lexai-platform\apps\api>

  (You should see "(venv)" at the start of the line)

SECTION 8 — CREATE ENVIRONMENT FILES (.env)
=========================================================================

You need TWO .env files — one for the API and one for the Web.

----------------------------------------------------------------------
8A. API environment file
----------------------------------------------------------------------

Navigate back to the API folder (apps/api) and create a file named ".env"

  > cd C:\Projects\lexai-platform\apps\api
  > copy NUL .env      (creates empty .env file on Windows)

Open .env in any text editor and paste this content,
replacing the DUMMY VALUES with your real keys:

-------- apps/api/.env --------
ENVIRONMENT=development
SECRET_KEY=your-random-secret-key-at-least-32-chars

SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.XXXXXXXXXX
SUPABASE_JWT_SECRET=your-supabase-jwt-secret

# Ollama local model (no API key needed!)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_CHAT_MODEL=llama3.1:8b
OLLAMA_EMBEDDING_MODEL=nomic-embed-text

# LangSmith — leave false unless you want to debug AI answers
LANGCHAIN_TRACING_V2=false
LANGCHAIN_API_KEY=
LANGCHAIN_PROJECT=lexai-dev

REDIS_URL=redis://localhost:6379

ALLOWED_ORIGINS=["http://localhost:3000"]
ALLOWED_HOSTS=["localhost","127.0.0.1"]
-------------------------------

----------------------------------------------------------------------
8B. Web (Next.js) environment file
----------------------------------------------------------------------

Navigate to apps/web and create a file named ".env.local"

  > cd C:\Projects\lexai-platform\apps\web
  > copy NUL .env.local

Open .env.local and paste this content:

-------- apps/web/.env.local --------
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.XXXXXXXXXX
NEXT_PUBLIC_API_URL=http://localhost:8000
--------------------------------------

  NOTE: NEXT_PUBLIC_SUPABASE_ANON_KEY is the "anon/public" key from
  Supabase (different from the SERVICE_ROLE_KEY used in the API).
  Find it in your Supabase project → Settings → API → Project API Keys.

SECTION 9 — SET UP THE DATABASE (Supabase)
=========================================================================

Your database migrations are in the supabase/migrations/ folder.
Run them in order inside Supabase SQL Editor:

  1. Go to https://supabase.com → your project → SQL Editor
  2. Open and run:  supabase/migrations/001_initial_schema.sql
  3. (optional) Run: supabase/seed.sql  to populate test data

  OR use Supabase CLI:

  > npm install -g supabase
  > supabase login
  > supabase link --project-ref YOUR_PROJECT_REF
  > supabase db push

  Dummy output:
    Applying migration 001_initial_schema.sql...✓
    Migration applied successfully.

SECTION 14 — TROUBLESHOOTING
=========================================================================

PROBLEM: pnpm not found
  FIX: Run  npm install -g pnpm@9.15.0  and restart your terminal

PROBLEM: "Activate.ps1 cannot be loaded because running scripts is disabled"
  FIX: Run this once in PowerShell as Administrator:
       Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

PROBLEM: Python packages fail to install (unstructured, sentence-transformers)
  FIX: Make sure Visual C++ Build Tools are installed:
       https://visualstudio.microsoft.com/visual-cpp-build-tools/

PROBLEM: Redis connection error
  FIX: Make sure Redis is running.
       Run Docker command:  docker run -d -p 6379:6379 --name redis redis:alpine

PROBLEM: Supabase errors in the API
  FIX: Double-check your apps/api/.env file has the correct Supabase keys

PROBLEM: Ollama connection error ("Connection refused" or "model not found")
  FIX 1: Make sure Ollama is running — open Task Manager and check, or run:
          ollama serve
  FIX 2: Make sure you pulled the models:
          ollama pull llama3.1:8b
          ollama pull nomic-embed-text
  FIX 3: Check OLLAMA_BASE_URL=http://localhost:11434 in your .env

PROBLEM: AI gives bad or inaccurate answers
  FIX: Try a better model — edit OLLAMA_CHAT_MODEL in apps/api/.env:
          OLLAMA_CHAT_MODEL=mistral          (good general purpose)
          OLLAMA_CHAT_MODEL=llama3.1:8b      (good reasoning)
          OLLAMA_CHAT_MODEL=gemma2:9b        (excellent for instructions)
       Then pull it:  ollama pull mistral

PROBLEM: Next.js shows blank page or 500 error
  FIX: Double-check apps/web/.env.local has NEXT_PUBLIC_SUPABASE_URL and
       NEXT_PUBLIC_SUPABASE_ANON_KEY set correctly
